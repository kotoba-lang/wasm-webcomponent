demo-cap.wasm (runtime has-capability? check)

demo-cap.wasm is the byte-for-byte output of kotoba wasm emit demo_cap.kotoba --policy demo_policy.edn in kotoba-lang/kotoba (declares a single has_capability(id) : i32 host import). Unlike examples/kgraph, this isn't about an effectful host call -- demo_cap.kotoba's main asks "do I have :notify/show?" at runtime and branches on the answer ((if (has-capability? :notify/show) (bump 6) 0)). src/has-capability.js answers that from a plain list of granted capability names -- this page grants notify/show, so the result is 7; deny it and the same bytes return 0 instead.

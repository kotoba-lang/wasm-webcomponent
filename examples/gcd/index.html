gcd.wasm (new .kotoba source, recursive, zero imports)

Unlike hello.wasm (a constant) or the kgraph example (host calls), gcd.wasm is a genuinely new, self-contained recursive algorithm written directly in .kotoba (see gcd.kotoba) and compiled with kotoba wasm emit gcd.kotoba --package-lock <empty-deps-lock> in kotoba-lang/kotoba (96 bytes). main() computes gcd(1071, 462) — the Euclidean algorithm's recursive case runs for real in the browser's own WebAssembly engine, not folded away at compile time.

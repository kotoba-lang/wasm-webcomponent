worker-http-post-demo (http_post via the SAB+COOP bridge, Worker-hosted)

Imports http_post. `Atomics.wait` (inside http-post-bridge.js's createSabHttpPostBridge) is disallowed on the main/DOM thread, so this guest is instantiated and called entirely inside a dedicated Worker (kotoba-wasm-worker-host.js), not this page's main thread -- see kotoba-wasm-worker-element.js. On its own page (not alongside index.html's other demos) so this Worker's own startup and module-graph loading (it pulls in the same vendored crypto library actor-host.js does) isn't competing with two OTHER WASM instantiations running on the same main thread at the same time. Also requires a cross-origin-isolated page (Cross-Origin-Opener-Policy: same-origin + Cross-Origin-Embedder-Policy: require-corp response headers -- see http-post-bridge.js's exported COOP_COEP_HEADERS) for SharedArrayBuffer to exist at all; opened without those headers (e.g. this static file served plainly), the guest fails with a clear error instead of hanging. The src defaults to ../http-post-echo.wasm (a fixed example target, http://example.test/echo -- won't actually resolve) but can be overridden with a ?src= query parameter, which test/browser/verify_http_post_browser.cljs uses to point at a fixture generated at test time with a real local echo server's URL baked in.

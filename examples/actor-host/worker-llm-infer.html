worker-llm-infer-demo (llm_infer via the SAB+COOP bridge, Worker-hosted)

Imports llm_infer. Like http_post (see worker-http-post.html), a synchronous host import cannot itself `await` a real network call, so `llm_infer` reuses the identical SharedArrayBuffer+Atomics.wait bridge -- kotoba-wasm-worker-host.js passes the SAME bridge instance as both httpPostBridge and llmInferBridge. `llm_infer` POSTs the raw prompt bytes to the llm-infer-url attribute and reads the raw completion text back -- that URL MUST be a developer-controlled proxy that itself holds any real LLM provider API key server-side, never a provider endpoint called directly with a key embedded in this page's own JS/HTML. The src defaults to ../actor-host-demo.wasm but, like the http-post demo, is overridden with a ?src= query parameter; llm-infer-url comes from ?llm-infer-url=, pointed at a real local test server by test/browser/verify_llm_infer_browser.cljs.

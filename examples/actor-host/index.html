actor-host-demo.wasm (kototama.contract's actor:host ABI, partial)

Imports now/log_append/sha256_hex (module "kotoba", same convention kotoba.wasm-exec and kototama.tender use). Wired via actor-host.js -- a browser-side port of kotoba-lang/kototama's kototama.contract (HostCaps/RuntimeLimits/ validate-import-surface, pre-flight AND per-call grant checks, same as the JVM tender). Only 4 of the 8 actor:host imports are implemented here -- gen-keypair/sign/ verify/http-post would need either an async Web Crypto/fetch call inside a host import (not possible without JS Promise Integration) or a hand-rolled synchronous crypto implementation (not attempted here) -- see src/actor-host.js's header comment for the honest scope note.
